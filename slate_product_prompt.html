Product Vision
Dailies shouldn't slow you down.
SLATE is a cross-platform production tool that ingests raw camera footage, automatically syncs audio, scores and sorts takes by quality without discarding anything, and assembles scene-based or subject-based rough cuts for review — all accelerated by AI. Built equally for scripted narrative and documentary workflows.

CORE PRINCIPLE Nothing is ever thrown away
  • Bad takes are deprioritized, not deleted. They remain accessible at all times with a single filter toggle.
  • AI scores are advisory — the editor, director, or producer always has final authority on take selection.
  • All original media is preserved and linked. SLATE only ever creates proxy references and metadata overlays.
  • Score reasoning is shown inline: "Focus soft at 0:02 — subject blinked" — not a black box.
Feature Specifications
What SLATE does, in detail.
Full feature set across ingest, AI processing, review, and export. Each spec block maps to a discrete development scope.
F-01Ingest & Media Management
  • Watch folder monitoring on local drives, network volumes (SMB/NFS), and cloud (S3/GCS/Backblaze B2)
  • Accepts all major camera formats: ARRIRAW, BRAW, ProRes, R3D, MP4/H.264, MXF
  • Auto-detects scene/slate/take from filename, metadata XMP, and embedded SMPTE timecode
  • Generates LUT-baked proxy (H.264 1080p) alongside original — originals are never touched
  • Checksum verification on ingest; alerts on corrupt files before processing begins
  • Batch import from camera cards and frame.io via API
F-02AI Audio Sync
  • Primary sync via clap detection + waveform cross-correlation (handles double-claps, missed slates)
  • Fallback to SMPTE timecode or Ambient/Ambient-compatible metadata
  • Multi-track ISO support: auto-assigns boom, lav channels with configurable mix preference
  • Drift correction over long takes (auto-conforms audio to camera sample rate)
  • Outputs polyphonic XML (FCPXML, AAF, OMF) for downstream delivery to post
  • Manual override: drag-sync in the waveform editor if AI sync misses
F-03Take Scoring & Ranking
  • Computer vision analysis: focus detection, exposure clipping, motion blur, lens flare, blocking errors
  • Audio analysis: clipping, background noise spikes, proximity, missing channels
  • Performance scoring (narrative mode): transcription + speech analysis for flub detection, energy, pacing
  • Documentary mode: content analysis scores based on subject clarity, interview intelligibility, emotional register
  • Each take receives a composite score (0–100) with per-dimension breakdown visible on hover
  • User can adjust scoring weights per project (e.g. prioritize audio over picture for dialogue scenes)
  • Filtered views: "Best take per setup," "All circled takes," "All takes" — default shows top 3 per scene
F-04Rough Cut Assembly
  • Narrative mode: assembles EDL/XML using highest-scored take per setup, organized by scene then shot
  • Documentary mode: assembles by subject/interview subject, or by theme/topic tags assigned during review
  • Director's cut mode: manual take selections override AI picks in the assembly
  • Output: FCPXML (Final Cut Pro X), EDL (Avid, Premiere), DaVinci Resolve XML
  • Assembly preview plays directly in SLATE before export — annotate and adjust before sending to editor
  • Version history: each assembly export is versioned and can be recalled or compared
F-05Review & Annotation
  • Frame-accurate commenting: click any frame, leave a text or voice note
  • Circle/X/Flag system maps to FCP/Avid metadata on export
  • Multi-user review with presence indicators (who's watching what right now)
  • Director approval workflow: takes move from "Pending" → "Reviewed" → "Approved" state
  • Shareable review links (time-limited, no login required for guest reviewers)
  • Mobile review: full playback, annotation, and approval on iPhone/iPad

AI Architecture
What's under the hood.
SLATE uses a multi-model AI pipeline. Each task routes to the right model — nothing is over-engineered. All AI outputs are transparent and overrideable.
AI-01Transcription Engine
  • WhisperX (local, on-device) for privacy-sensitive productions; hosted Whisper API as optional cloud fallback
  • Speaker diarization: automatically labels who is speaking (Interview A, B, etc.) for documentary transcripts
  • Transcripts are searchable, exportable as SRT/WebVTT, and synced to timecode for paper-cut workflows
AI-02Vision Analysis
  • Frame sampling at configurable intervals (default: 2fps) to detect technical quality issues
  • Focus map: uses face detection + depth estimation to score subject sharpness vs background
  • Exposure analysis: histogram-based clipping detection, flags both over and under-exposure
  • Motion analysis: camera shake, unwanted blur, subject tracking confidence
  • Models: YOLO v8 (object detection), DepthAnything v2 (depth), custom classifier trained on film QC datasets
AI-03Performance & Content Scoring
  • Narrative: cross-references script text (if provided) against transcript — flags flubs, missed lines, ad-libs
  • Prosody analysis: speech energy, pacing, pauses — helpful signal for emotional performance quality
  • Documentary: semantic analysis of interview content to surface thematically rich moments
  • All scoring is optional and can be disabled per project. Models run locally on Apple Silicon via CoreML / MLX
AI-04Audio Sync AI
  • Primary: FFT-based waveform cross-correlation — handles noisy sets, double-claps, missed slates
  • Secondary: SMPTE timecode reader (LTC and VITC) as ground-truth fallback
  • Confidence score per sync: green (high confidence), yellow (manual verify recommended), red (manual required)
  • Drift monitoring: checks sync at regular intervals during long takes, auto-corrects or flags drift
Technical Architecture
How it's built.
SLATE is designed to run processing locally on Apple Silicon with optional cloud sync. All media stays on set — only metadata and proxies are ever sent to the cloud.

Swift / SwiftUI Core ML / MLX AVFoundation Core Data Metal
  • Native macOS app optimized for Apple Silicon (M1/M2/M3/M4). All AI models run on-device via MLX.
  • GPU-accelerated proxy transcode using VideoToolbox and Metal Performance Shaders
  • Background daemon handles watch-folder monitoring even when app is closed
  • Menu bar icon with progress indicators and quick-access shortcuts
PLATFORMMobile (iOS/iPadOS)
Swift / SwiftUI AVKit CloudKit Push Notifications
  • Optimized for review, annotation, and approval — not media processing (handled by desktop)
  • Offline mode: proxies cached locally for review on set without WiFi
  • iPad split-screen: notes and playback side by side
PLATFORMWeb (Review Portal)
Next.js 14 React HLS.js tRPC Vercel
  • Lightweight review portal — no install required for external reviewers
  • HLS adaptive streaming for proxy delivery from cloud storage
  • Real-time collaboration via WebSockets (comments appear live for all viewers)
  • Secure link generation with expiry, password-protection, and permission tiers
BACKENDCloud Infrastructure
Supabase Cloudflare R2 WhisperX PostgreSQL Redis
  • Metadata and collaboration sync via Supabase Realtime (PostgreSQL + WebSockets)
  • Proxy storage on Cloudflare R2 (S3-compatible, egress-free) — originals never leave production storage
  • Optional self-hosted deployment for studios with strict data sovereignty requirements

# SLATE — AI Dailies Processing & Review Platform
## Product Summary
Build SLATE, a cross-platform production tool (macOS desktop, iOS mobile, web portal) 
that automates the dailies processing pipeline for film and television production.
It handles both scripted narrative and documentary workflows.

## Core Principles
- Never delete or trash media. Bad takes are deprioritized, not removed.
- All AI decisions are transparent, explainable, and overrideable by humans.
- Originals are never modified. SLATE works with proxies and metadata only.
- The system must work offline on set (desktop + mobile).

## Key Features

### 1. Media Ingest
- Watch folder monitoring (local drives, SMB/NFS network volumes, S3/GCS cloud storage)
- Support for ARRIRAW, BRAW, ProRes, R3D, MXF, MP4/H.264
- Auto-detect scene/slate/take from filenames, XMP metadata, SMPTE timecode
- GPU-accelerated proxy generation (H.264 1080p with optional LUT bake)
- Checksum verification on ingest
- Frame.io API import support

### 2. AI Audio Sync
- Primary: waveform cross-correlation clap detection (handles missed/double slates)
- Fallback: SMPTE LTC/VITC timecode
- Multi-track ISO support with auto-channel assignment (boom, lav)
- Drift correction for long takes
- Manual drag-sync override in waveform editor
- Export: FCPXML, AAF, OMF with synced multi-track audio

### 3. Take Scoring (AI)
- Vision: focus, exposure, motion blur, camera shake, lens flare detection
- Audio: clipping, noise spikes, proximity, missing channels
- Narrative mode: transcription + flub detection against uploaded script
- Documentary mode: semantic content scoring, interview intelligibility
- Composite score 0–100 with per-dimension breakdown shown to user
- User-configurable scoring weights per project
- Score reasoning shown inline (e.g. "Focus soft at 0:02 — subject blinked")
- Filtered views: Best take per setup / Circled takes / All takes (default: top 3)

### 4. Rough Cut Assembly
- Narrative mode: EDL/XML assembled by scene, highest-scored take per setup
- Documentary mode: assemble by interview subject or user-defined themes/topics
- Director override: manual take selections replace AI picks in assembly
- Export: FCPXML, EDL, DaVinci Resolve XML
- Assembly preview playback within SLATE before export
- Version history for all assembly exports

### 5. Review & Annotation
- Frame-accurate timestamped comments (text and voice)
- Circle / X / Flag system (maps to NLE metadata on export)
- Multi-user presence (see who's watching in real time)
- Director approval workflow: Pending → Reviewed → Approved
- Shareable guest review links (time-limited, no login required)
- Mobile full review, annotation, and approval

### 6. Documentary Mode (Additional)
- Footage organized by subject / shooting day, not scene/slate
- WhisperX transcription with speaker diarization
- Full text search across all transcribed footage
- Clip tagging by topic/theme keywords
- Assemble rough cut by dragging theme bins into an outline
- Paper cut export to NLE

## Platform Targets

Desktop (macOS — Primary):
- Native Swift/SwiftUI, Apple Silicon optimized
- All AI models run on-device via Core ML / MLX (no cloud required for AI)
- GPU-accelerated transcode via VideoToolbox + Metal
- Background daemon for watch-folder monitoring
- Menu bar icon with progress and quick access

Mobile (iOS/iPadOS — Review):
- Swift/SwiftUI, AVKit playback
- Proxy cached locally for offline review on set
- Push notifications when dailies are ready
- iPad split-screen: notes + playback side by side

Web (Review Portal — Lightweight):
- Next.js 14 / React
- HLS adaptive streaming from cloud proxy storage
- Real-time comments via WebSockets
- Secure link generation (expiry, password, permission tiers)
- No install required for external reviewers

## Backend
- Supabase (PostgreSQL + Realtime for collaboration sync)
- Cloudflare R2 for proxy storage (originals never leave production storage)
- Optional self-hosted deployment for data-sovereign studios
- WhisperX for local transcription; hosted Whisper API as cloud fallback

## Out of Scope (V1)
- Editing within SLATE (export to NLE only)
- Color grading or VFX review
- Audio mixing or dialogue editing
- Windows or Android support

## Success Metrics
- Time from card wrap to director review link: under 30 minutes for a 10-hour shoot day
- Audio sync accuracy: >98% correct on first attempt
- Take scoring correlation with editor's final picks: >80%
- Producer/director net promoter score: >60 after first production
